Movie API

This API is the server-side component of a movies web applicaiton that will allow users to access information about different movies, directors, and genres.
Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints

Request URL HTTP Method Request Data Format Response Data Format
List All Movies /movies GET None A JSON object holding a complete list of movies in the database.
Return Data About a Single Movie /moves/[Title] GET None A JSON object holding data about a single movie. Example:
{
 Title: "Howl's Moving Castle",
 Description: "description text here",
 Genre: {
  Name: "Fantasy"
  Description: "description text here"
 }
 Director: {
  Name: "Hayao Miyazaki",
  Bio: "bio text here",
  Birth: "date"
 }
 Actors: ["two featured actors"],
 ImagePath: "howlsmovingcastle.png"
 Featured: "Yes"
}

/[Name]		 GET None A JSON object holding data about a genre. Example:
{
 Genre: “Fantasy”,
 Description: “description text here”

/[Name]		 GET None A JSON object holding data about a director. Example:
{

 Bio: “biography text here”,

Allow New Users to Register /users POST A JSON object holding data about the user registering a new account. Example:
{
 Username: "Kelly",
 Password: "postgres,"
 Email: "kellys@mail.com",
 Birthday: "1990-09-09"
} 		A JSON object with the new user data and a text message indicating that the user has successfully registered.
Allow Users to Update Their User Information /users/[username] PUT A JSON file holding data to update the user information. Example:
{
 Username: "Kelly",
 Password: "postgres,"
 Email: "kellys@mail.com",
 Birthday: "1990-09-09"
} 		A JSON object with the new user data and a text message indicating that the user has successfully updated their username.

/movies/[MovieID]		 POST None A text message showing that the movie has successfully been added to their list of favorite movies.
